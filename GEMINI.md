# 네이버 부동산 매물 수집기 (Naver Real Estate Scraper) 명세서

이 문서는 프로젝트의 전반적인 구조와 핵심 로직을 설명하여, AI 에이전트(Gemini)가 프로젝트의 문맥을 빠르게 파악하고 유지보수할 수 있도록 돕기 위해 작성되었습니다.

## 1. 프로젝트 개요
*   **주요 파일:** `naver_real_estate_scraper.py`
*   **기능:** 사용자가 전국 법정동 단위로 지역을 검색/선택하면, 네이버 부동산 API를 통해 해당 지역의 아파트 단지 목록과 매물 정보(매매, 전세, 월세 등)를 수집하여 CSV 파일로 저장합니다.
*   **주요 라이브러리:** `scrapling` (요청 및 파싱), `pandas` (엑셀 데이터 처리), `tkinter` (GUI), `zipfile`, `json`

## 2. 핵심 아키텍처 및 클래스

### 2.1. `LegalDongScraper` (법정동 데이터 수집기)
*   **역할:** 프로그램이 외부 깃허브 저장소에 의존하지 않고 스스로 최신 행정안전부 법정동 데이터를 구축하도록 합니다.
*   **소스:** 행정표준코드관리시스템 (`code.go.kr`)의 ZIP 압축 파일(내부 XLSX 파일).
*   **로직:**
    *   `fetch_latest_data()`: 공식 URL에서 ZIP 파일을 다운로드하고 메모리 내에서 압축을 풀어 엑셀 파일을 읽습니다.
    *   `_parse_df(df)`: `pandas`를 사용해 '현존'하는 지역만 추출하고, 주소 문자열을 분석하여 `[시/도]`, `[시/군/구]`, `[읍/면/동/리]` 계층으로 정확히 분리합니다.

### 2.2. `ScraperGUI` (사용자 인터페이스 및 오케스트레이션)
*   **역할:** Tkinter 기반의 사용자 UI 제공 및 데이터 수집 스레드 관리.
*   **주요 기능:**
    *   `init_data()`: 프로그램 시작 시 `legal_dong_data_full.json` 파일의 생성일을 검사합니다. 30일 이상 지났거나 데이터 개수가 부족(10,000개 미만)하면 `LegalDongScraper`를 호출해 자동 갱신합니다.
    *   `search_region()`: "거여", "괴안" 처럼 동 이름의 일부나 전체 주소를 입력해도 유연하게 매칭되도록 텍스트를 정규화하여 검색합니다.
    *   `start_scraping()`: UI 응답성을 위해 메인 스레드와 분리된 백그라운드 데몬 스레드에서 수집(`run_scraper`)을 시작합니다.
    *   `ThreadSafeConsole`: 터미널의 `stdout`/`stderr` 출력을 GUI의 ScrolledText 위젯으로 안전하게 리다이렉션합니다.

## 3. 핵심 수집 로직 (Scraping Functions)

*   **`safe_request(session, url, headers)`**
    *   **목적:** 네이버의 429 (Too Many Requests) 차단을 우회하고 안정성을 확보합니다.
    *   **특징:** 브라우저와 동일한 보안 헤더(`sec-ch-ua`, `User-Agent` 등)를 삽입하며, 지수 백오프(Exponential Backoff)를 적용해 최대 5회까지 재시도합니다.
*   **`get_token(session, headers)`**
    *   **목적:** 네이버 부동산 API 호출에 필요한 Bearer 토큰을 메인 페이지에서 추출합니다.
    *   **특징:** `window.App` 형태와 `__NEXT_DATA__` JSON 형태 두 가지 구조를 모두 파싱할 수 있는 견고한 정규표현식을 사용합니다.
*   **`get_naver_cortar_no(...)`**
    *   **목적:** 사용자가 선택한 지역명(예: "서울특별시")과 네이버 API가 반환하는 지역명(예: "서울시") 사이의 불일치를 해결하여 정확한 지역 코드(cortarNo)를 찾습니다.
    *   **특징:** 시/도 매칭 시 앞 2글자(예: "서울", "경기")를 비교하는 로직을 사용하여 정확도를 높였으며, 하위 계층도 단계별로 유연하게 탐색합니다.
*   **데이터 파이프라인:** `fetch_apartments` (단지 목록 조회) -> `fetch_listings_for_complex` (단지별 매물 페이징 조회) -> `save_to_csv` (거래 방식별 분리 저장).

## 4. 유지보수 및 확장 시 주의사항
*   **수정 원칙:** 네이버 API나 웹페이지 구조(`window.App` 등)가 변경되어 토큰 획득이나 데이터 조회가 실패할 경우, 가장 먼저 브라우저 개발자 도구를 통해 응답 구조를 확인하고 `get_token` 또는 `get_page_text` 로직을 수정해야 합니다.
*   **법정동 데이터:** `LegalDongScraper`의 엑셀 파싱 로직(`_parse_df`)은 "경기도 부천시 소사구 괴안동" 같은 4단계 주소를 올바르게 시/도, 시/군/구, 동/리로 매핑하도록 설계되어 있습니다. 이 구조가 깨지지 않도록 유의하세요.
