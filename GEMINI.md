# 네이버 부동산 매물 수집기 (Cloudtype Full-Stack Edition) 명세서

본 프로젝트는 프론트엔드와 백엔드 모두 Cloudtype(서울 리전)을 통해 배포되는 통합 웹 애플리케이션입니다.

## 1. 시스템 아키텍처
*   **통합 호스팅 (Cloudtype):** 단일 서버 환경에서 Next.js 프론트엔드와 Python Flask 백엔드가 공존하거나, Cloudtype의 멀티 서비스 기능을 통해 상호 연결됩니다. 서울 리전의 이점을 활용하여 네이버 부동산 API와의 초고속/고안정성 통신을 보장합니다.
*   **데이터 자동화 (GitHub):** GitHub Actions가 매시간 `dong.json`(지역)과 `token.json`(인증)을 최신화하여 서버의 연산 부하를 줄이고 수집 시작 속도를 극대화합니다.

## 2. 소스 코드 구조 (Line Range Index)

### 2.1. `server.py` (통합 백엔드)
*   **1-15:** 라이브러리 임포트 및 Flask 설정
*   **17-30:** Scrapling 응답 텍스트 추출 유틸리티
*   **32-55:** `/api/token` - 실시간 토큰 획득 엔드포인트
*   **57-80:** `/api/complexes` - 단지 목록 조회 엔드포인트
*   **82-110:** `/api/listings` - 상세 매물 조회 엔드포인트
*   **112-116:** 서버 실행 설정 (Port 5000)

### 2.2. `src/app/page.tsx` (웹 인터페이스)
*   **1-25:** 타입 정의 및 유틸리티 함수
*   **27-60:** 상태 관리(State) 변수 선언
*   **62-75:** 초기 데이터 로딩 및 로그 함수
*   **77-130:** 수집 프로세스 오케스트레이션 (Token -> Complexes -> Listings)
*   **132-155:** XLSX 기반 엑셀 내보내기 로직
*   **157-350:** Tailwind CSS 기반 반응형 UI 렌더링

### 2.3. `update_legal_dong.py` (데이터 업데이터)
*   **1-15:** 라이브러리 임포트
*   **17-45:** 네이버 토큰 선제적 획득 로직
*   **47-90:** 행안부 공식 XLSX 기반 법정동 파싱 클래스
*   **92-110:** 메인 실행부 (파일 저장 및 종료 제어)

## 3. 운영 가이드
*   **배포 환경:** Cloudtype Python 서비스로 배포하며, `requirements.txt`에 명시된 모든 의존성이 설치되어야 합니다.
*   **보안:** `scrapling` 세션을 통해 브라우저 지문을 관리하며, 서울 IP를 사용하여 네이버의 해외 IP 차단 정책을 원천적으로 회피합니다.
*   **갱신:** 지역 정보가 바뀌거나 토큰이 만료될 경우 GitHub Actions가 자동으로 처리하며, 수동 갱신은 GitHub Actions 탭에서 수행합니다.
